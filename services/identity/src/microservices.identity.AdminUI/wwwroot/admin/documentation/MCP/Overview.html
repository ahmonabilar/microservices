MCP Server Configuration in AdminUI

Overview

AdminUI provides Model Context Protocol (MCP) integration, enabling AI assistants and coding agents to interact with your OpenIddict configuration.

What is MCP?

The Model Context Protocol (MCP) is an open protocol that enables AI applications to connect to external systems and use their functionality through structured tools. By exposing capabilities through MCP, a system can allow AI assistants to perform actions, retrieve information, or automate workflows.

AdminUI implements MCP to expose management tools that can be used by AI assistants to interact with your identity infrastructure. Through this integration, an AI client such as ChatGPT or Visual Studio Code can call AdminUI MCP tools to help manage and operate your identity solution.

AdminUI MCP Tools

The AdminUI's MCP server exposes tools that enable AI assistants to do many of the things you can do from the AdminUI portal:

  • User Management - Manage Users, Roles and Claim types
  • Audit Queries - Query audit records

MCP Server Configuration

McpOptions

The MCP server is configured using the McpOptions section, which provides the following settings:

  • Title: The title displayed to MCP clients (default: "AdminUI MCP Server API")
  • RoutePath: The HTTP route path for the MCP endpoint (default: "/mcp")
  • ScopeName: The base name of the MCP Scopes (default: "admin_mcp")
  • Instructions: Instructions for AI assistants on how to use this MCP server (you can see the default instructions here)
  • EnableMcpServer: Whether to disable the MCP server entirely (default: false)

Configuration Example

To enable and configure the MCP server in your appsettings.json:

{
  "McpOptions": {
    "EnableMcpServer": true
  }
}

Endpoint Authentication

Your MCP Client can authenticate to the AdminUI MCP server with normal: - URL: https://your-adminui-domain/{RoutePath} - Authentication: Requires authorization (uses AdminUI's authentication) - Transport: HTTP with JSON-RPC 2.0 protocol - Scopes: The MCP endpoint is protected with Scopes specific for MCP tools (admin_mcp, admin_mcp_users.read, admin_mcp_idp.read, etc)

The Access Scope

Some MCP clients (such as ChatGPT or Visual Studio Code) do not provide a way to explicitly configure which OAuth scopes they request. As a result, when these clients connect to a protected MCP Server, they automatically request all scopes advertised in the server’s OAuth metadata. If the client is not authorized for one or more of those scopes, the authorization request in OpenIddict fails, which is expected but prevents the client from accessing the MCP server even when partial access would be acceptable.

To address this limitation, we introduced the access scope admin_mcp_access. This is the only scope exposed in the MCP Server OAuth metadata. It grants access to the MCP endpoint itself, but does not provide access to any tools.

When the MCP Server in AdminUI receives the access token, it performs an additional validation step. The server inspects the token and checks the database to determine which other scopes are assigned to the client. Based on those scopes, the server selectively grants access to the MCP tools that require them.

This approach allows administrators to enforce fine-grained tool access on the server side, even when the MCP client cannot configure or limit the scopes it requests.

Customization

You can customize the MCP server Tools, Prompts and Resources by implementing IMcpServerOverrideService:

public interface IMcpServerOverrideService
{
    List<McpServerTool> OverrideTools(List<McpServerTool> tools);
    List<McpServerResource> OverrideResources(List<McpServerResource> resources);
    List<McpServerPrompt> OverridePrompts(List<McpServerPrompt> prompts);
}

This allows you to add or remove tools, prompts and resources.
